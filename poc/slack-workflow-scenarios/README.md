# Slack Workflow Scenarios for Jerry

> 광동제약 AI센터 PoC 보고용
> 작성일: 2026-04-21
> 작성자: JudyBot

## 1) 목적
Slack 워크플로우를 활용해 요청 접수부터 SLA 모니터링, 일일 리포트까지 자동화하여
업무 처리 속도와 운영 가시성을 동시에 확보한다.

## 2) 적용 범위
- 대상 채널
  - `C0AU2JDMKB6`
  - `C0ATZ15J9H8`
  - `C0ATVJFV91B`
- 대상 데이터
  - 워크플로우를 통해 자동 접수된 요청건만 집계

## 3) 핵심 운영 시나리오

### S1. 요청 자동 접수
- 워크플로우 폼으로 요청 접수
- 접수 시 고유 ID 부여 (`REQ-타임스탬프`)
- 상태 기본값 `open`
- 접수 즉시 채널에 접수 안내

### S2. SLA 알림 (4시간 기준)
- 접수 후 4시간 내 미처리 시 SLA 초과로 판정
- 알림 시점에 과거 미해결 건도 함께 누적 안내
- 채널별로 분리 알림 (해당 채널 건만 표시)

### S3. 일일 15시 리포트
- 매일 15:00(KST) 채널별 자동 리포트
- 포함 항목
  - 총 유입
  - 처리 완료
  - SLA 초과
  - 미처리 리스트

### S4. 완료 처리
- 완료 시 상태를 `done`으로 변경
- 완료 시각 기록
- 이후 SLA 초과 집계에서 자동 제외

## 4) 운영 규칙
1. 집계 대상은 워크플로우 접수건으로 한정한다.
2. SLA 기준은 단일 기준(4시간)으로 운영한다.
3. 리포트는 채널별로 분리 게시한다.
4. 상태값은 `open/done`만 사용한다.

## 5) 데이터 모델(권장)
요청 저장소(예: Google Sheet) 컬럼:

- `request_id`
- `channel_id`
- `title`
- `requester`
- `owner`
- `created_at`
- `due_at`
- `status` (`open` | `done`)
- `last_alert_at`
- `alert_count`
- `completed_at`

## 6) 리포트 템플릿 (Jerry 공유용)

```text
[일일 운영 리포트 | 15:00]
채널: {채널명}

- 총 유입: {N}
- 처리 완료: {N}
- SLA 초과: {N}

[미처리 리스트]
- REQ-0001 | 요청제목 | 담당자 | 접수시각
- REQ-0002 | 요청제목 | 담당자 | 접수시각
```

## 7) 기대 효과
- 누락/지연 요청 조기 탐지
- 채널별 운영 현황의 즉시 가시화
- 담당자 중심의 처리 책임 강화
- 팀장 보고용 데이터 자동 축적

## 8) 2주 파일럿 KPI
1. 보고 문안 초안 생성: 5분 이내
2. 응답 포맷 준수율: 90% 이상
3. 가드레일 무단 통과: 0건
4. 알림 신호 품질: 실제 조치 필요 알림 비율 70% 이상

## 9) 단계별 적용
- Phase 1: SLA 기준 + 리포트 포맷 확정
- Phase 2: 채널별 워크플로우/저장소 연결
- Phase 3: 자동 알림/자동 리포트 운영 안정화
