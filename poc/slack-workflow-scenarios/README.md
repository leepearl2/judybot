# Slack Workflow Scenarios for Jerry

> 광동제약 AI센터 PoC 보고용
> 작성일: 2026-04-21
> 작성자: JudyBot

## 1) 목적
Slack 워크플로우를 활용해 요청 접수부터 SLA 모니터링, 일일 리포트까지 자동화하여
업무 처리 속도와 운영 가시성을 동시에 확보한다.

## 2) 적용 범위
- 대상 채널
  - `C0AU2JDMKB6`
  - `C0ATZ15J9H8`
  - `C0ATVJFV91B`
- 대상 데이터
  - 워크플로우를 통해 자동 접수된 요청건만 집계

## 3) 오늘 진행한 작업 요약 (2026-04-21)
- 워크플로우 기반 요청 접수 구조 설계 및 시범 적용
- 메일 수신 이벤트를 Slack 채널로 전달하는 자동화 플로우 검증
- Google Drive 파일 변경/공유 이벤트를 Slack에서 확인하는 플러그인 연동 테스트
- 채널별 운영 분리(요청 집계/리포트 분리) 기준 확정

## 4) 핵심 운영 시나리오

### S1. 요청 자동 접수
- 워크플로우 폼으로 요청 접수
- 접수 시 고유 ID 부여 (`REQ-타임스탬프`)
- 상태 기본값 `open`
- 접수 즉시 채널에 접수 안내

### S2. SLA 알림 (4시간 기준)
- 접수 후 4시간 내 미처리 시 SLA 초과로 판정
- 알림 시점에 과거 미해결 건도 함께 누적 안내
- 채널별로 분리 알림 (해당 채널 건만 표시)

### S3. 일일 15시 리포트
- 매일 15:00(KST) 채널별 자동 리포트
- 포함 항목
  - 총 유입
  - 처리 완료
  - SLA 초과
  - 미처리 리스트

### S4. 완료 처리
- 완료 시 상태를 `done`으로 변경
- 완료 시각 기록
- 이후 SLA 초과 집계에서 자동 제외

### S5. 전자결재 실적용 테스트 시나리오
- 전자결재 요청 발생 시 워크플로우로 자동 접수
- 상태값(`open -> done`) 기반으로 처리 현황 추적
- 결재 지연 건은 SLA 기준(4시간)으로 자동 알림 대상 편입
- 일일 15시 리포트에 전자결재 요청의 유입/완료/미처리 현황 반영

### S6. 메일 플러그인 연동 시나리오
- 메일 수신(또는 특정 조건 필터링) 이벤트를 Slack 채널에 자동 전달
- 전달 메시지에 핵심 메타데이터(제목/발신자/수신시각/요청유형) 포함
- 운영자는 Slack 내에서 우선순위 판단 후 요청 처리 플로우로 연계

### S7. Drive 플러그인 연동 시나리오
- Drive 내 파일 생성/수정/공유 이벤트를 Slack에서 알림 수신
- 문서 링크, 변경 유형, 변경자 정보를 함께 표기
- 변경 이벤트를 업무 요청과 연결해 추적 가능하도록 구성

## 5) 운영 규칙
1. 집계 대상은 워크플로우 접수건으로 한정한다.
2. SLA 기준은 단일 기준(4시간)으로 운영한다.
3. 리포트는 채널별로 분리 게시한다.
4. 상태값은 `open/done`만 사용한다.

## 6) 데이터 모델(권장)
요청 저장소(예: Google Sheet) 컬럼:

- `request_id`
- `channel_id`
- `title`
- `requester`
- `owner`
- `created_at`
- `due_at`
- `status` (`open` | `done`)
- `last_alert_at`
- `alert_count`
- `completed_at`

## 7) 리포트 템플릿 (Jerry 공유용)

```text
[일일 운영 리포트 | 15:00]
채널: {채널명}

- 총 유입: {N}
- 처리 완료: {N}
- SLA 초과: {N}

[미처리 리스트]
- REQ-0001 | 요청제목 | 담당자 | 접수시각
- REQ-0002 | 요청제목 | 담당자 | 접수시각
```

## 8) 기대 효과
- 누락/지연 요청 조기 탐지
- 채널별 운영 현황의 즉시 가시화
- 담당자 중심의 처리 책임 강화
- 팀장 보고용 데이터 자동 축적

## 9) 2주 파일럿 KPI
1. 보고 문안 초안 생성: 5분 이내
2. 응답 포맷 준수율: 90% 이상
3. 가드레일 무단 통과: 0건
4. 알림 신호 품질: 실제 조치 필요 알림 비율 70% 이상

## 10) 단계별 적용
- Phase 1: SLA 기준 + 리포트 포맷 확정
- Phase 2: 채널별 워크플로우/저장소 연결
- Phase 3: 자동 알림/자동 리포트 운영 안정화
