04 핵심 인사이트
1
로그인은 했지만 절반은 "구경꾼"이다
로그인 155명 중 1~5회에 그친 사용자가 26명(17%)이에요. 평균 14.9회 로그인이지만 중간값은 12회 — 소수 헤비유저가 평균을 끌어올리고 있어요. Drive 실 데이터 보유자 34명(18%), Chat 활동자 59명(31%)에 불과해요. 계정 배포 성공과 실사용 성공은 완전히 다른 문제예요. 2차에서는 "접속 횟수"가 아닌 "편집·생성·공유 건수"를 측정 기준으로 바꿔야 해요.
측정 기준 전환 필요
2
DM 72% — Slack을 카카오톡처럼 쓰고 있다
총 메시지 6,554건 중 DM이 4,721건(72.0%), 공개 채널은 118건(1.8%)에 불과해요. DAU 최고 97명을 기록한 4월 15일에도 실제 메시지를 보낸 사람은 53명(55%)뿐이었어요 — 나머지 44명은 접속만 하고 읽기만 했어요. 활성 사용자의 절반 가까이가 생산자가 아닌 소비자에 머물러 있어요. Slack AI 채널 요약이 이 구조를 깰 열쇠예요. 채널에 올릴 이유가 생기면 읽을 이유도 따라와요.
생산자 전환 필요
3
연휴 이후 사용량 급락 — 습관이 형성되지 않았다
4월 20~29일 피크 구간(DAU 79~90명, 메시지 397~670건)에서 5월 연휴 이후 DAU 2~9명으로 급락했어요. 5월 5일 복귀 이후에도 43명에 그쳤고 파일 업로드는 8건이었어요. 습관이 형성됐다면 연휴 뒤에도 복귀가 빨라야 해요. 아직 Slack이 "없으면 불편한 도구"가 아닌 "있으면 쓰는 도구"예요. 2차에서는 격주 미션으로 사용 빈도를 강제 유지하는 구조가 필수예요.
습관 형성 실패
4
MS Office 파일이 여전히 Drive를 점령 — Gemini 활용의 걸림돌
Drive 이벤트 중 GWS 네이티브 문서(Sheets·Docs·Slides)는 15,175건(10.4%), MS Office 파일은 7,198건(4.9%)이에요. 비율상 2:1이지만 MS Office 파일을 GWS에 올려서 여는 방식이 여전히 많아요. Gemini는 Google 문서에서만 완전히 동작해요. MS Office 중심 워크플로우가 유지되는 한 AI 기능 체감이 어렵고, 사례 발굴도 제한돼요. 2차에서는 신규 문서를 Google 문서로 만드는 것부터 유도해야 해요.
워크플로우 전환 필요
5
Slack ↔ GWS 연결이 시작됐다 — 통합 시나리오의 씨앗
OAuth 로그에서 Slack이 GWS Drive를 호출한 횟수가 1,715건이고 연동 사용자는 36명이에요. Claude 연동도 이미 5명이 사용 중이에요. 아직 소수지만 Slack ↔ Drive ↔ Gemini ↔ Claude로 이어지는 통합 워크플로우의 씨앗이 심어졌어요. 2차에서 이 연결고리를 격주 미션으로 의도적으로 확장하면 사용 사례가 가장 풍부하게 나올 수 있어요.
통합 시나리오 가능
05 2차 PoC 운영 방안
PRE 시작 전 — 환경 셋업 완료 후 진입 필수

1차에서 로그인 실패 296건의 주요 원인이 2SV 미설정이었어요. 2차는 시작 전에 접속 환경을 완전히 잡고 들어가야 첫 주부터 이탈이 없어요.

계정 발급 후 킥오프 전날까지 GWS 로그인 + 2SV 설정 완료 요청 (사전 안내 문자 발송)
킥오프 당일 미완료자 현장에서 즉시 처리 — 완료 확인 후 세션 시작
Slack 알림 설정 현장 완료 — PC·모바일 둘 다. 알림 없으면 안 들어와요
팀 공유 드라이브 폴더 사전 생성 — 킥오프 때 바로 써볼 수 있게 준비
01 킥오프 — 팀별 맞춤 실습 1시간 1일차

팀 전체가 함께 하는 1시간 실습이에요. 설명 10분, 실습 50분으로 구성하고 AI센터가 직접 진행해요. 1차에서 "가이드 문서만 줬을 때"보다 "같이 해봤을 때" 체감이 훨씬 빠랐어요.

10분 — 오늘 쓸 것만 소개: Slack AI 채널 요약, Gemini, NotebookLM 각 1분 데모
20분 — Slack 실습: 채널 만들기 → 팀원 초대 → Slack AI로 채널 요약 직접 눌러보기
20분 — GWS 실습: 공유 드라이브 폴더에 문서 업로드 → Gemini로 요약 요청해보기
10분 — NotebookLM: 팀 문서 1개 업로드 후 질문 1개 해보기 — 이것만 해도 충분
팀 담당자 1명 지정 — 이후 AI센터와 소통 창구, 사례 공유 채임자
02 AI 도구 단계별 활용 로드맵 — 주차별 집중 기능 AI 활용 핵심

한꺼번에 다 쓰라고 하면 아무것도 안 써요. 주차별로 하나씩 집중해서 써보게 유도해요. MS Office → GWS 네이티브 전환도 이 로드맵에서 자연스럽게 유도해요.

1~2주차 · Slack AI
채널 요약으로 "오늘 뭐 올라왔나" 확인 습관 만들기
AI 검색으로 지난 대화·파일 찾기 실습
허들 노트 — 회의 후 요약 자동 저장 체험
3~4주차 · Gemini in Workspace
Gmail: Gemini로 메일 초안 작성 / 긴 메일 요약
Google 문서: 회의록·보고서 초안 생성
Sheets: 데이터 분석 질문, 수식 자동 생성
5~6주차 · NotebookLM
팀 내부 문서 5개 이상 업로드 → Q&A
SOP·매뉴얼 기반 챗봇처럼 활용
오디오 오버뷰로 문서 브리핑 청취
7~8주차 · 통합 워크플로우
Slack → Drive → Gemini 연결 실무 시나리오
Gem(맞춤형 AI) 팀 전용 1개 만들어보기
사례 정리 후 전사 공유 — PoC 마무리
03 채널 운영 정책 — DM 72%를 깨는 구조 운영 규칙

규칙만으로는 안 바뀌어요. 채널에 올리면 편하고, 채널을 보면 유익한 구조를 만들어야 해요. Slack AI 채널 요약이 이 구조의 핵심이에요.

AI센터 역할: 주 2회 #poc-2차 채널에 꿀팁·사례 업로드. 채널을 볼 이유를 먼저 만들어요
팀 담당자 역할: 주 1회 팀 채널에 "이번 주 써본 것" 한 줄 공유 (의무 아닌 습관화 유도)
Slack AI 채널 요약을 매일 아침 루틴으로 — "채널 보는 게 귀찮다"는 인식 깨기
질문은 DM 아닌 채널에 — 같은 질문 두 번 안 받고, 답변이 팀 자산이 돼요
목표: 공개 채널 비율 1.8% → 15% 이상 (8주 내)
04 격주 체크인 — 데이터로 보고 바로 개입 모니터링

1차엔 중간 개입 체계가 없었어요. 2차에선 격주로 데이터를 보고 안 쓰는 팀에 바로 넛지를 줘요. 한 달 지나면 패턴이 굳어져요.

격주 확인 항목 (10분이면 됨): GWS 관리자 콘솔 Drive 활성 사용자 수 / Slack 분석 탭 DAU·채널 비율
2주차 신호 기준: 로그인 주 3회 미만 or Drive 편집 0건 → 담당자에게 Slack DM 직접 넛지
4주차 중간 체크인: 3문항 Google 폼 설문 — "써본 기능 / 막힌 지점 / 다음에 써보고 싶은 것"
설문 결과로 다음 2~4주 미션 조정 — 팀마다 속도가 다를 수 있어요
우수 팀은 Gem 만들기·Workspace 공유 드라이브 심화 기능 추가 안내
05 사례 공유 — 조건이자 자산 필수 조건

2차 PoC의 핵심 조건이에요. 동시에 이 사례들이 전사 도입 제안서의 핵심 근거가 돼요. 허들을 낮추는 게 전부예요 — 형식은 완전 자유, 3줄이면 충분해요.

3줄 템플릿: "어떤 업무에 / 어떤 기능을 / 시간이 얼마나 줄었나(또는 어떤 효과)" — Slack 채널에 바로 올리기
AI센터가 매달 사례 취합 → Slack Canvas에 누적 정리
월 1회 "이달의 사례" 하이라이트 — 잘 쓴 팀 명시, #전체공지 게시로 전사 동기 부여
PoC 종료 시 사례집 정리 → 팀별 Before/After 정량 효과 문서화 → 본 도입 근거로 직행
