GWS + Slack PoC 검증기간 기능별 상세 체크리스트

기간: 4/1 ~ 5/27 · 목적: 실사용 운영 가능성 + 보안/통제 + 확산 가능성 검증

사용 방법: 기능 단위로 주 단위 점검하고, 체크 완료 항목은 담당자·완료일·증빙 링크(문서/로그)까지 함께 남긴다. 체크 상태는 같은 브라우저에서 자동 저장된다.

1) 계정·접근관리 기초 안정화

  • 1-1.사용자 계정 목록 최신화 (입/퇴사·조직이동·겸직 계정 반영)
  • 1-2.GWS 그룹 ↔ Slack 유저그룹 매핑 검증 (누락/중복/오배정 확인)
  • 1-3.신규 사용자 온보딩 절차 테스트 (초대 → 권한부여 → 첫 로그인까지 리드타임 측정)
  • 1-4.권한 회수 절차 테스트 (퇴사/이동 사용자 접근 차단 + 공유자산 소유권 이관)
  • 1-5.예외 권한 요청 프로세스 정리 (승인자, 기준, 만료일, 정기 재검토 주기)

2) 채널·워크스페이스 운영 협업 구조

  • 2-1.PoC 공식 채널 구조 확정 (공지/운영문의/파일럿 피드백 채널 분리)
  • 2-2.채널 네이밍/설명/고정메시지(가이드, 문의 방식) 표준화
  • 2-3.필수 알림 정책 설정 (멘션 기준, 공지 권한, 노이즈 최소화 룰)
  • 2-4.운영 이슈 접수 템플릿 배포 (발생시간/재현경로/스크린샷/영향도)
  • 2-5.주간 공지 리듬 운영 (월: 변경사항, 수: 활용팁, 금: FAQ/개선사항)

3) 파일·문서 협업 (GWS Drive/Docs) 실무 사용성

  • 3-1.공유드라이브 구조 점검 (업무영역별 분리, 관리자/소유 책임 명시)
  • 3-2.권한 기본값 적용 확인 (Viewer/Commenter 기본, Editor 승인제)
  • 3-3.문서 공동편집 시나리오 테스트 (동시편집, 버전복구, 코멘트/승인 흐름)
  • 3-4.파일 공유 실수 방지 장치 확인 (외부공유 경고, 링크 접근 범위 검증)
  • 3-5.파일명/폴더명/버전명 컨벤션 공지 및 준수율 확인

4) JudyBot/자동화 기능 PoC 차별 포인트

  • 4-1.핵심 명령어/기능 목록 확정 (사용자에게 공개할 기능과 제한 기능 분리)
  • 4-2.대표 사용 시나리오 5개 테스트 (요약, 공지 초안, 체크리스트, Q&A, 보고 초안)
  • 4-3.오답·할루시네이션 대응 가이드 배포 (검증 필요 문구/책임소재/재질문 방식)
  • 4-4.장애 대응 기준 수립 (응답 지연, 실패, 권한오류 시 수동 대체 프로세스)
  • 4-5.기능 개선 요청 백로그 운영 (요청내용, 우선순위, 반영일, 반영결과 공개)

5) 보안·통제 온보딩 전 필수

  • 5-1.외부공유 차단/제한 정책 재검증 (OU·그룹·드라이브별 예외 확인)
  • 5-2.다운로드·인쇄·복사 제한 테스트 (파일유형/권한레벨별 결과 기록)
  • 5-3.DLP 시나리오 테스트 (민감 키워드 업로드/외부 공유 시 탐지·차단 확인)
  • 5-4.감사로그 리뷰 루틴 수립 (일/주 단위 점검자, 임계치, 경보 처리 절차)
  • 5-5.정보보호팀 공유 문서 업데이트 (모니터링 가능/불가 범위와 보완책 분리 명시)

6) 사용자 온보딩·지원 체감 품질

  • 6-1.사용자 가이드 2종 배포 (요약본 1p + 상세 매뉴얼)
  • 6-2.첫 주 필수 교육/안내 세션 운영 (라이브 데모 + Q&A)
  • 6-3.문의 SLA 공지 (예: 긴급 4h, 일반 24h) 및 실제 준수율 측정
  • 6-4.반복 문의 TOP10 FAQ로 전환하여 고정메시지/문서에 즉시 반영
  • 6-5.파일럿 사용자 만족도 체크 (주간 펄스 설문 + 개선 액션 연결)

7) 성과 측정·보고 의사결정

  • 7-1.PoC KPI 확정 (활성 사용자, 협업량, 처리시간, 문의건수, 정책위반 건수)
  • 7-2.주간 리포트 템플릿 통일 (성과·이슈·리스크·다음주 액션)
  • 7-3.1분기 결과 정리 (계획 대비 변경점/실행률/학습 포인트)
  • 7-4.2분기 확장 계획 수립 (월별 목표, 필요 리소스, 리스크 대응)
  • 7-5.확대/유지/중단 판단표 작성 (효과, 리스크, 운영비용, 조직 수용성)

작성 목적: PoC 기간 중 기능별 실행 상태를 일관되게 추적하고, 최종 의사결정에 필요한 근거를 축적하기 위함
